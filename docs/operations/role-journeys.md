# Role Journeys (P3.1)

## Resident

1. Signs in via email/password or magic link
2. Completes onboarding wizard (role, profile, optional team invites)
3. Dashboard shows: case counts, program goals progress, recent activity
4. Logs a new case: selects template → fills fields → submits for approval
5. Edits drafts, views approval status, resubmits rejected cases
6. Views own analytics (case volume, specialty distribution, duty hours)
7. Uses mobile app for bedside case logging with offline draft support

## Supervisor

1. Signs in, sees pending approvals dashboard
2. Reviews case entries: reads field values, checks completeness
3. Approves or rejects with required comment
4. Views team analytics, identifies gaps in resident experience
5. Manages duty hour attestations

## Director

1. All supervisor capabilities plus:
2. Manages case templates (create/edit/version fields)
3. Sets program goals and targets
4. Configures evaluation forms and milestone frameworks
5. Views program-wide analytics and compliance reports
6. Manages faculty evaluations

## Institution Admin

1. Configures tenant settings, SSO, webhooks, AI, payment gateway
2. Manages users (invite, role changes, deactivation)
3. Sets data retention policies
4. Views audit logs and billing
5. Manages storage quotas and accreditation frameworks

## Platform Admin

1. Cross-tenant management via platform admin tools
2. Views system health, audit logs, billing across tenants
3. Configures subscription plans
4. Break-glass access for tenant support
