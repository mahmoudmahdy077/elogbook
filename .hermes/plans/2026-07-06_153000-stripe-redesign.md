# E-Logbook Enterprise — Stripe-Inspired Design Upgrade Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Transform E-Logbook from a functional dark-themed app into a premium, Stripe-inspired enterprise SaaS with refined typography, layered surfaces, and blue-tinted depth.

**Prototype reference:** http://194.146.13.223/elogbook-redesign.html

**Architecture:** Replace the HeroUI-styled design tokens (teal/indigo, boxy panels, weight-600 headings) with a Stripe-inspired system (deep navy surfaces, warm purple accent #6C63FF, weight-300 headings, blue-tinted multi-layer shadows, conservative 4-10px radii). Update `@elogbook/shared` design tokens, then propagate through web components. Keep mobile aligned with same tokens.

**Tech Stack:** TypeScript (strict), Next.js 16, Tailwind CSS v4, Framer Motion, HeroUI (gradually phase out for custom components)

## Risks & Open Questions

1. **HeroUI dependency** — The current codebase uses HeroUI components throughout. Full removal is a separate project. Scope this plan to: update tokens + globals.css + key components. HeroUI-styled elements will look slightly off until they're replaced.  
2. **Light theme** — The prototype is dark-only. The light theme mappings need testing after token changes.  
3. **Mobile sync** — WatermelonDB models reference old token values? Verify.  
4. **Backward compat** — The token shape change (adding `surface.raised`, `surface.overlay`, changing `border.DEFAULT` value) may cause TS errors in files that access nested token paths.  
5. **Do we remove Framer Motion?** The prototype uses CSS animations (lighter). Consider replacing Framer Motion with CSS keyframes for dashboard only — keep Framer for complex interactions (case form wizard, approval animations).
